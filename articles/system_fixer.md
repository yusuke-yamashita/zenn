# System Fixer (システムフィクサー)
System Fixerとは、ソフトウェア、組織、人間関係、文化的文脈を一つの巨大な「システム」と捉え、そこに潜む不整合や断絶（バグ）を解消し、機能不全に陥った現場を再起動させる専門職です。

QAエンジニアとしての目的志向、テストエンジニアとしての観察眼と、アジャイルコーチや組織開発ファシリテーターの領域を横断するスキルを用い、システムが持続的に価値を生み出すための「修正（Fix）」を行います。

[System Fixerに込めた思い（背景と経緯）はこちら](https://zenn.dev/55_ymzn/books/yamazun_introduction/viewer/14_passion_behind_system_fixer)

# 概要と定義
System Fixerは、単なるトラブルシューティングや現状復帰（マイナスをゼロに戻すこと）を目的としません。以下の2つの思想に基づき、システムを再構築します。
- 金継ぎ（Kintsugi）: 壊れた箇所や不具合の歴史をあえて残しつつ、新しい価値を接合することで、以前よりも強固で味わい深いシステムへと昇華させます。
- 生成的（Generative）な場の醸成: クリストファー・アレグザンダーの思想に基づき、システムを機械的な統制下に置くのではなく、内側から自律的に秩序や価値が生まれる「生成的」な状態へ導きます。直した後も、予期せぬ良さが勝手に育っていくような「生きている場」を作ります。

## 特徴
- テスター視点: 違和感や不穏な空気といった定性的な事象を「バグ」として言語化し、早期発見する能力に特化しています。
- 原点思考: 既存のフレームワークや「正解」を当てはめるのではなく、「そもそも何のためにそのシステムが存在するのか」という原点（本質的価値）に立ち返ります。その原点を尊重しつつ、価値を発揮させるためであれば、慣例にとらわれない手段で介入します。
- 内省的:答えを急いで提示するのではなく、クライアントと共に「なぜその状態になっているのか」を深く省みるプロセスを重視します。システムの一部としての自らの立ち振る舞いや影響力も含め、常に問い続ける静かな姿勢を貫きます。

# 対象領域と得意とするフェーズ
1. ターゲットとなる組織状況
**「ベンチャー体質から大企業体質への変貌期」**にあるIT企業において最も力を発揮します。 
急速な拡大に伴い、創業時の熱量が失われたり、部門間のセクショナリズム（断絶）が発生したりしている変革期において、硬直化したシステムに風穴を開けます。

2. 介入する4つのシステムレイヤー
System Fixerは、以下のレイヤーを複合的に観察し、修正パッチを当てます。

- コンピューターシステム: ソフトウェアの構造的欠陥。
- 組織のシステム: 意思決定プロセスの歪みや権限の不均衡。
- 関係性のシステム: メンバー間の不信感、忖度、心理的安全性の欠如。
- 文脈のシステム: 組織文化や、そのコミュニティが持つ熱狂のメカニズム。

# System Fixerの行動指針（Code of Conduct）
System Fixerは、システムのルールをハックし、時に破壊的な手段を用いることもありますが、唯一絶対の倫理規定を持っています。

**「人を自己実現の対象として扱わない」**

修正の対象はあくまで「システム（構造・関係性）」であり、「人間そのもの」ではありません。
個人の尊厳を傷つけたり、人を道具としてコントロールすることは固く禁じています。むしろ、システムによって圧殺されかけている個人の尊厳を守るために活動します。
